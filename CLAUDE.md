# CLAUDE.md — FlareGuard Edge Project Guidance

## Architecture & Conventions
- **Runtime**: Cloudflare Workers (V8 Serverless Environment).
- **Core Principle**: Strict privacy-first. API tokens and Zone IDs must only exist in-memory during request execution.
- **Storage Model**: Stateless by default. Optional persistent storage (D1/KV/R2/Queue) activates only when bindings are present in `wrangler.toml`. All storage bindings are commented out by default.
- **Privacy Guarantees** (enforced in `src/api/routes.js` and `src/db/index.js`):
  - API tokens are **NEVER** stored, logged, or passed to any storage function.
  - Raw Zone IDs and Account IDs are **NEVER** stored — only a 16-char SHA-256 hash is persisted as a lookup key.
  - All DB/KV/R2 calls are guarded: `if (hasDB(env))` / `if (env.CACHE)` / `if (env.REPORTS)`.
- **Compliance Mapping**: Map all new AI-Gateway and Workers AI findings to NIST AI Risk Management Framework (AI RMF) or OWASP Top 10 for LLMs controls.

## Directory Structure
```
src/
  index.js                  — Worker entry point + router + queue consumer
  api/routes.js             — REST handlers (privacy enforcement lives here)
  audit/
    engine.js               — Unified baseline-driven audit engine
    evaluators/
      zone-setting.js       — Zone settings (SSL, TLS, HTTPS, security level…)
      waf.js                — WAF / OWASP CRS checks
      dnssec.js             — DNSSEC status
      bot.js                — Bot Fight Mode / Bot Management
      rate-limit.js         — Rate limiting rules
      access.js             — Zero Trust / Access MFA + IdP checks
      workers.js            — Zombie worker detection, secret env var scan
  cache/index.js            — KV cache wrapper (no-op when CACHE not bound)
  db/index.js               — D1 helpers (no-op when DB not bound)
  storage/index.js          — R2 report storage (no-op when REPORTS not bound)
  queue/index.js            — Queue producer + consumer
  ui/dashboard.js           — Single-file SPA served by the Worker
  utils/
    cf-api.js               — Cloudflare API client
    privacy.js              — hashId() for one-way ID hashing
    uuid.js                 — crypto.randomUUID() wrapper
migrations/
  0001_init.sql             — D1 schema (zone_id_hash, never raw IDs)
baseline.yaml               — Security check definitions (YAML)
```

## API Endpoints
| Method | Path | Description |
|--------|------|-------------|
| GET  | `/` | Dashboard SPA |
| GET  | `/api/status` | Feature flags + privacy attestation |
| POST | `/api/test-connection` | Validates credentials via live Cloudflare API call |
| POST | `/api/audit/zone` | Run a zone security audit |
| POST | `/api/audit/account` | Enqueue account-wide scan (or run sync if no Queue) |
| GET  | `/api/history/:zoneId` | Audit history (requires D1) |
| GET  | `/api/audit/:auditId/findings` | Detailed findings for one audit (requires D1) |
| GET  | `/api/drift/:zoneId` | Drift between two most recent audits (requires D1) |
| GET  | `/api/report/:key` | Download stored JSON report (requires R2) |

## Development & Test Commands
- Install: `npm install`
- Dev Server: `npm run dev` (fully stateless, no bindings required)
- Apply local D1 migrations: `npm run db:migrate:local`
- Lint: `npm run lint`
- Test: `npm test`

## Adding New Security Checks
1. Add a check entry to `baseline.yaml` with a unique `id`, `service`, `severity`, and `nist_controls`.
2. If the service is new, create `src/audit/evaluators/<service>.js` and add the import + case to `src/audit/engine.js`.
3. Never hardcode Cloudflare API URLs — use `CloudflareAPI` methods from `src/utils/cf-api.js`.
4. Map findings to NIST controls (or NIST AI RMF / OWASP LLM Top 10 for AI-related checks).

## Strict "Do Not Do" List
- Never store API tokens or raw Zone/Account IDs in any persistent layer.
- Never write audit results to an external log aggregator.
- Never hardcode Cloudflare API endpoints — use the `CloudflareAPI` class.
- Do not append "Generated by Claude Code" to git commits or PR descriptions.
- Do not enable storage bindings in `wrangler.toml` by default — keep them commented out.
